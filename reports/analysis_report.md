# Windows Wireshark Endpoint Traffic Analysis Report

## 1. Executive Summary

This lab captured and analyzed network traffic generated by a Windows endpoint using Wireshark. The investigation focused on DNS queries, encrypted web traffic metadata, local discovery protocols, and endpoint-to-router communication.

The analysis confirmed that modern endpoint traffic can be investigated without exposing private content. Most web activity is encrypted, but metadata such as protocols, ports, DNS queries, packet timing, and local discovery behavior remains useful for SOC-style analysis.

## 2. Objective

The objective was to document how normal endpoint traffic appears in Wireshark and to identify protocol behavior relevant to beginner SOC and network analysis work.

## 3. Scope

Only traffic visible from the analyst's own Windows endpoint was analyzed. The lab does not include third-party user monitoring, password capture, traffic decryption, or unauthorized network inspection.

## 4. Methodology

1. Selected the active Windows Wi-Fi adapter in Wireshark.
2. Captured endpoint traffic during normal browsing and local network activity.
3. Used display filters to isolate DNS, encrypted traffic, and discovery protocols.
4. Reviewed packet list and Info column evidence.
5. Sanitized screenshots before publishing.
6. Documented findings in a structured report.

## 5. Filters Used

```wireshark
ip.addr == <endpoint_ip>
dns
dns.qry.name contains "youtube"
dns.qry.name contains "whatsapp"
tcp.port == 443 || udp.port == 443
ssdp
dns || mdns || nbns || ssdp
```

## 6. Observations

### 6.1 DNS Queries

DNS packets showed domain lookup behavior generated by the endpoint. DNS query and response packets were visible for domains opened during the capture.

Evidence:

- [`dns-youtube-query.png`](../screenshots/dns-youtube-query.png)
- [`dns-whatsapp-query.png`](../screenshots/dns-whatsapp-query.png)

Analyst note:

DNS is useful because it shows which domains an endpoint attempted to resolve, but DNS visibility may be affected by DNS caching or encrypted DNS.

### 6.2 Encrypted Traffic Metadata

Traffic over port 443 was observed. This indicates encrypted web/app communication. The packet content is not readable, but metadata such as protocol, port, timing, and packet length remains useful.

Evidence:

- [`encrypted-traffic-port-443.png`](../screenshots/encrypted-traffic-port-443.png)

Analyst note:

Modern web traffic is generally encrypted. Network analysts should focus on metadata, destination behavior, volume, timing, and anomalies instead of expecting plaintext content.

### 6.3 SSDP Local Discovery

SSDP M-SEARCH packets were observed. This indicates the endpoint was performing local service discovery using multicast communication.

Evidence:

- [`ssdp-msearch-local-discovery.png`](../screenshots/ssdp-msearch-local-discovery.png)

Analyst note:

SSDP is common in local networks and may be triggered by Windows, browsers, media/casting features, or UPnP-aware applications.

### 6.4 Router/Device Discovery Response

SSDP response traffic from the local network was observed. This shows endpoint-to-router or endpoint-to-local-device discovery communication.

Evidence:

- [`router-ssdp-response.png`](../screenshots/router-ssdp-response.png)

### 6.5 Combined Local Discovery Protocols

The combined discovery filter showed DNS, SSDP, and other local name resolution/discovery protocols in the same capture.

Evidence:

- [`local-discovery-combined.png`](../screenshots/local-discovery-combined.png)

## 7. Findings

| Finding | Meaning |
|---|---|
| DNS query packets were visible | Endpoint requested domain-to-IP resolution |
| Port 443 traffic was present | Endpoint generated encrypted web/app traffic |
| SSDP M-SEARCH packets were visible | Endpoint performed local service discovery |
| Local discovery protocols appeared | Normal Windows/browser local network behavior was present |
| Raw content was not readable | Modern HTTPS/TLS traffic protects private content |

## 8. Security Relevance

This lab is useful for security operations because endpoint traffic analysis helps analysts understand:

- What domains an endpoint resolved
- Whether traffic is encrypted or unencrypted
- Which protocols are present
- Whether local discovery activity is normal or unusual
- How to document network observations without exposing sensitive data

## 9. Limitations

- The capture was endpoint-level only.
- Encrypted web content was not decrypted.
- DNS visibility may be reduced if DNS-over-HTTPS or caching is used.
- The lab does not represent full-network monitoring.

## 10. Conclusion

The lab demonstrates practical Wireshark usage for endpoint traffic analysis on Windows. It shows how to capture, filter, interpret, and document DNS, encrypted traffic metadata, and local discovery protocols in a privacy-safe and professionally structured way.
