# LinkedIn Post Draft

I recently completed a hands-on **Windows Wireshark Endpoint Traffic Analysis Lab** focused on ethical packet capture and protocol analysis.

The objective was not to monitor an entire Wi-Fi network, but to analyze traffic generated by my own Windows endpoint and document what normal network behavior looks like from a SOC analyst perspective.

In this lab, I worked on:

- Capturing traffic from the correct Windows Wi-Fi adapter
- Filtering endpoint traffic in Wireshark
- Analyzing DNS queries for visited domains
- Reviewing encrypted traffic metadata over port 443
- Understanding TCP/UDP traffic behavior
- Identifying SSDP local discovery traffic
- Recognizing mDNS, NBNS, and LLMNR-style local name resolution
- Separating useful metadata from encrypted/private content
- Writing privacy-safe technical documentation

Key learning:

Modern traffic is mostly encrypted, so Wireshark is not about reading private website content. The real analyst skill is understanding metadata, protocols, ports, DNS behavior, packet timing, endpoint communication, and local discovery patterns.

This project helped me strengthen my fundamentals in:

- Wireshark
- Network Traffic Analysis
- DNS Analysis
- TCP/UDP
- HTTPS/TLS metadata
- Local discovery protocols
- SOC-style documentation
- Ethical cybersecurity practice
