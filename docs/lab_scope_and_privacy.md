# Lab Scope and Privacy

## Scope

This lab was performed from a Windows endpoint using Wireshark. The purpose was to understand normal network behavior generated by the endpoint itself.

The analysis focused on:

- DNS lookups generated by the endpoint
- Encrypted web traffic metadata
- Local network discovery protocols
- Endpoint-to-router communication
- Wireshark filtering and protocol interpretation

## What Was Not Done

This lab does not include:

- Monitoring other users' private traffic
- Password capture
- Session/token capture
- Traffic decryption
- Unauthorized scanning
- Man-in-the-middle activity
- Any attempt to bypass encryption

## Privacy Controls Applied

Before publishing the evidence, screenshots were sanitized by redacting:

- Raw packet detail panes where unnecessary identifiers may appear
- MAC address visibility in lower packet details
- Unnecessary packet bytes
- Unneeded local network identifiers

The public repository keeps the focus on analysis skill, protocol understanding, and documentation rather than exposing environment-specific details.

## Correct Interpretation

This project demonstrates endpoint-level traffic analysis, not full network monitoring. A normal user connected to a Wi-Fi network usually sees their own device traffic, broadcast/multicast discovery traffic, and selected local network protocol activity.

## Professional Use Case

This type of lab is useful for:

- SOC analyst fundamentals
- Network troubleshooting fundamentals
- DNS analysis practice
- Understanding encrypted traffic metadata
- Learning normal Windows endpoint network behavior
- Building packet-analysis documentation skill
