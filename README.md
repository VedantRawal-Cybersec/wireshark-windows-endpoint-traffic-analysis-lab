# Windows Wireshark Endpoint Traffic Analysis Lab

A practical network traffic analysis lab performed on a Windows endpoint using Wireshark. The project focuses on ethical packet capture, DNS inspection, encrypted traffic metadata, local discovery protocols, and structured SOC-style documentation.

## Objective

Analyze traffic generated by a single Windows endpoint and document how common protocols appear inside Wireshark during normal network usage.

The lab focuses on:

- Endpoint traffic filtering
- DNS query analysis
- Encrypted traffic metadata over port 443
- Local discovery protocols such as SSDP, mDNS, NBNS, and LLMNR
- Router and endpoint communication
- Packet evidence documentation
- Privacy-safe reporting

## Scope

This lab analyzes only traffic visible from the analyst's own Windows endpoint. It does not attempt to intercept, decrypt, or inspect third-party user traffic.

Sensitive values such as full device identifiers, MAC addresses, and unnecessary raw packet details have been redacted from screenshots before publishing.

## Key Skills Demonstrated

- Capturing traffic from the correct Windows network adapter
- Filtering endpoint traffic using display filters
- Identifying DNS lookups and DNS responses
- Understanding why HTTPS content is encrypted
- Recognizing TCP/UDP port 443 traffic
- Identifying local discovery protocols
- Reading packet list, packet details, and Info columns
- Separating useful metadata from private content
- Writing a structured analysis report

## Main Filters Used

```wireshark
ip.addr == <endpoint_ip>
dns
dns.qry.name contains "youtube"
dns.qry.name contains "whatsapp"
tcp.port == 443 || udp.port == 443
ssdp
mdns
nbns
llmnr
dns || mdns || nbns || ssdp
```

More filters are documented in [`filters/wireshark_filters.md`](filters/wireshark_filters.md).

## Findings Summary

- DNS queries revealed domain lookup behavior before web connections were established.
- Port 443 traffic showed encrypted web communication metadata without exposing page contents.
- SSDP traffic showed local UPnP/service discovery behavior using multicast communication.
- mDNS/NBNS/LLMNR traffic showed local name resolution and device discovery behavior on Windows.
- The capture confirms that modern endpoint analysis focuses on metadata, protocol behavior, timing, and communication flow rather than reading encrypted content.
